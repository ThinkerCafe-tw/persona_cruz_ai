# 🌌 00. 專案願景與 LibreChat Fork 核心理由

## 1. 專案願景：打造真正的個性化 AI 助手

本「個性化 AI 助手計畫」的核心願景是：**創造一個不僅僅是工具，更是能夠深度理解用戶、具備長期記憶、並能與用戶共同成長的智能夥伴。**

我們致力於：

*   **深度個性化**：AI 助手應能適應不同用戶的溝通風格、偏好和需求，提供真正量身定制的交互體驗。
*   **長期記憶**：助手需要有能力記住與特定用戶的歷史互動關鍵信息、用戶的明確指示以及個人背景資料，從而在持續的交流中提供更有上下文、更連貫的回應。
*   **無縫整合**：助手應能方便地嵌入到用戶現有的工作流程或業務平台中，成為日常任務的得力幫手。
*   **可信賴與可控**：用戶應對其 AI 助手的記憶內容和行為有清晰的認知和一定的控制力。

## 2. 為何 Fork LibreChat：站在巨人的肩膀上

根據「LibreChat 深度技術與產品評估報告」的全面分析，我們做出堅決的戰略決策：**Fork LibreChat 作為本計畫的基礎框架。**

**核心理由如下：**

*   **顯著的成本效益與風險降低**：
    *   LibreChat 提供了一個功能極其豐富、架構成熟且高度可擴展的平台。其價值遠遠超過從零開始開發的成本。
    *   預估將 LibreChat 改造為滿足我們需求的 MVP（最小可行產品）所需開發投入約為 **11-18 人週**。
    *   相比之下，從頭構建一個同等級別平台所需時間可能超過一年，成本和風險都將高出一個數量級。

*   **成熟的基礎設施，讓我們專注核心創新**：
    LibreChat 已經為我們解決了大量複雜且耗時的基礎設施問題，使我們能夠將寶貴的工程資源集中在打造產品的核心競爭力和差異化優勢上——即「個性化」與「長期記憶」。已解決的基礎設施包括：
    *   一個功能強大的、支援多供應商的**通用 AI 網關**。
    *   一套企業級的**多用戶身份驗證系統**。
    *   一個完整且即插即用的**語音互動框架** (STT/TTS)。
    *   一套成熟的、基於 Docker 的**部署與 CI/CD 流程**。
    *   一個基於現代技術棧 (React, shadcn/ui)、高度可客製化的**前端 UI**。
    *   一個可供擴展的、用於實現長期記憶的 **RAG 系統基礎**。

*   **戰略靈活性與無廠商鎖定**：
    *   LibreChat 對接極其廣泛的 AI 模型供應商（OpenAI, Azure, Anthropic, Google, AWS Bedrock 等），並支援自訂端點接入任何 OpenAI 相容 API 的服務（如自訓練模型、Ollama 等）。
    *   這徹底消除了被單一 AI 供應商鎖定的風險，為我們長期的 AI 戰略發展提供了堅實保障。

*   **活躍的開源社群與持續的項目演進**：
    *   LibreChat 是一個活躍的開源專案，擁有持續的更新和社群支持。
    *   其發展藍圖（如計畫中的「用戶特定記憶」功能）與我們的目標方向有一定程度的契合。

**總而言之，選擇 Fork LibreChat 不僅是一個可行的選項，更是加速我們「個性化 AI 助手計畫」實現的最佳催化劑。它讓我們能夠站在巨人的肩膀上，更快地實現創新，同時有效管理開發風險。**

接下來的設計文檔將詳細闡述我們如何在 LibreChat 的基礎上，逐步構建我們獨特的個性化 AI 助手。
