🌌 思考者咖啡：AI-人類協作進化藍圖

此藍圖旨在建立一套「活文檔驅動式協作」流程，該流程對 Rhaenyra 而言是「結構清晰、策略穩固」的，對 Leo 而言則是「無感、專注、流暢」的。核心是將制度隱藏於無形的自動化之中。

工作流程：從「想法」到「成果」的旅程

💡階段一：概念與結構化 (Rhaenyra 的產品設計院)

一切始於 Rhaenyra 在 cpo/tasks/ 下創建一個任務文檔 (e.g., TASK-001.md)。這是想法的播種之地。

🏔️土: 自動提供標準化的「產品規格模板」，讓 Rhaenyra 無需從零開始，專注於策略思考。
💧水: 在模板中自動注入「事前驗屍 (Pre-Mortem)」質疑清單，將未來風險轉化為當前的清晰邊界。

🤖階段二：精煉與賦能 (AI 團隊的自動化中樞)

當任務文檔被提交時，系統自動介入，進行精煉與賦能，確保其成為一份「即時可開發」的任務。

⚔️金: 透過 GitHub Action 自動檢查文檔格式，確保 `owner`, `status` 等元數據完整，實現無感知的治理。
🔥火: 在文檔中自動填充技術細節，如 API 端點、程式碼骨架和測試清單，為 Leo 準備好「開發燃料包」。

💻階段三：開發與心流 (Leo 的工程師聖殿)

任務以完美狀態出現在 engineering/tasks/ 中。Leo 打開文檔，看到的是清晰的賽道，可以直接進入心流模式進行開發。

此階段的目標是讓 AI 的工作完全透明，使 Leo 感覺不到流程的存在，眼中只有程式碼和需要征服的挑戰。

🌳階段四：開花與結果 (團隊共享的成就森林)

當 Leo 將任務標記為完成時，系統自動將成果展示出來，讓每一次努力都被看見。

🌱木: 自動更新根目錄的 `README.md`，將完成的任務加入「🏆 最新成果」展廳，並以視覺化方式更新「專案森林」的生長進度。

總結：一個共生的生態系統

這個流程創造了一個閉環：Rhaenyra 的策略思考，透過 AI 的自動化賦能，無縫轉化為 Leo 的高效開發，其成果又透過 AI 的自動化展示，反過來驗證了 Rhaenyra 的策略，並激勵整個團隊。每個人都在自己最擅長的領域工作，同時又能清晰地看見自己的貢獻如何推動整體前進。
